Hi Ankush (and It's FOSS team),

I'm writing from the Mailspring project. Your "Top 12 Best Email Clients for Linux" article is one of the top results people find when choosing a Linux email client, and we really appreciate Mailspring's inclusion.

I wanted to share some major updates we've shipped since the article was last updated in June 2025, in case a 2026 refresh is on the roadmap:

**Full calendar support (the big one):**
Mailspring now includes a complete CalDAV calendar — month, week, day, and agenda views, event creation/editing with server sync, drag-to-reschedule, recurring event support with exceptions, and calendar search. This was by far the most-requested missing feature.

**Linux-first improvements:**
- **Native Wayland support** via Electron 39 — no more XWayland
- **ARM64 (aarch64) builds** now available
- **Official AUR package** — the community package is now maintained officially
- System tray fixes for GNOME, Unity, and Ubuntu 25 (no more "•••" icon)
- Dark/light system tray icon support
- Default thin window framing with hamburger menu
- Broadened Fedora compatibility (libtidy soname handling)
- SMTP EHLO/HELO IPv6 support
- Linux Do Not Disturb detection

**Other highlights:**
- Grammar checking in the composer (LanguageTool, disabled by default)
- One-click unsubscribe (RFC 2369/8058)
- Comprehensive accessibility overhaul
- VCard import/export
- TypeScript v3→v5 and React upgrades
- Sync engine stability: iCloud infinite sync fix, memory leaks resolved, SQLite upgraded

The article currently notes some concerns about "spotty customer support" — we've been actively working to address this with faster response times and better community engagement. We'd love to discuss this further if it would help update that section.

Happy to provide screenshots, a walkthrough, or anything else that would be useful for an update. Thanks for everything you do for the Linux community.
